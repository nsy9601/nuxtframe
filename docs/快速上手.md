# 快速上手指南
## 一、环境搭建
### 1. 依赖安装
```bash
# 安装依赖
pnpm install

# 启动开发环境
pnpm dev
2. 环境要求
Node.js ≥ 20.x
pnpm ≥ 8.x
Nuxt ≥ 4.2.2
二、目录说明
plaintext
nuxt-ui-tag-admin/
├── app/                  # 核心代码目录
│   ├── components/       # 组件目录
│   │   ├── layout/       # 布局组件
│   │   ├── business/     # 业务组件
│   │   └── common/       # 通用组件
│   ├── composables/      # 组合式函数
│   ├── layouts/          # 布局文件
│   ├── pages/            # 页面文件（自动生成路由）
│   ├── assets/styles/    # 样式文件
│   ├── types/            # TS类型定义
│   └── utils/            # 工具函数
├── components/           # 通用模型页面组件
├── docs/                 # 项目文档
├── nuxt.config.ts        # Nuxt配置文件
└── package.json          # 依赖配置
三、新增业务模型
1. 新增页面文件
在 app/pages/ 目录下创建模型文件夹及页面文件：
plaintext
app/pages/
├── [模型名]/
│   ├── index.vue        # 列表页
│   ├── create.vue       # 新增页
│   ├── edit/[id].vue    # 编辑页
│   └── detail/[id].vue  # 详情页
2. 页面配置示例（列表页）
vue
<script setup lang="ts">
const modelName = '模型标识'; // 与后端模型名一致
</script>

<template>
  <ModelList :model-name="modelName" />
</template>
3. 后端配置要求
模型需包含 UIConfig 字段，配置模型级全局参数
业务字段需配置 ui 标签，遵循协议规范
实现协议约定的核心接口（getuitag、getlist、add、edit、del、getpermissions）
四、接口切换
1. 虚拟数据与真实接口切换
javascript
// 方式1：修改 nuxt.config.ts
runtimeConfig: {
  public: {
    useMockData: false // true=虚拟数据，false=真实接口
  }
}

// 方式2：页面内手动切换
import { setUseMockData } from '@/utils/apiSwitch';
setUseMockData(false); // 切换到真实接口
2. 缓存清理
javascript
// 清理 UI 配置缓存
import { clearUIConfigCache } from '@/composables/useUIConfig';
clearUIConfigCache('模型名'); // 清理指定模型缓存
clearUIConfigCache(); // 清理所有缓存
五、主题配置
1. 切换预设主题
通过顶部导航的「主题切换」按钮，选择预设主题（蓝色专业、紫色科技、绿色生态、橙色活力、红色警示、暗色模式）
2. 自定义主题
修改 app/assets/styles/themes.less 文件，新增或修改主题变量：
less
.theme-custom {
  --color-primary: #你的主色;
  --color-primary-light: #你的浅色;
  // 其他变量...
}
